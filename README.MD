# üìá Flashcard Manager

A full‚Äëstack flashcard management web application with a **dark theme**, built with **Python (Flask)** on the backend and **vanilla HTML/CSS/JavaScript** on the frontend.  
It supports **LaTeX** rendering (via MathJax), **Markdown** formatting in descriptions, **automatic RTL/LTR** text direction, **categorized flashcards** with full CRUD, and a fully **responsive** layout for desktop and mobile.

All frontend libraries are **served locally** ‚Äì no CDN dependencies. A Python script downloads them automatically.

---

## ‚ú® Features

- **Category Management** ‚Äì Create, view, and delete folders (categories) to organize your flashcards.
- **Flashcard CRUD** ‚Äì Add, edit, delete, and flip cards with a single click.
- **LaTeX Support** ‚Äì Content wrapped in `$...$` is rendered as mathematical formulas using MathJax.
- **Markdown in Descriptions** ‚Äì The back side supports GitHub‚Äëflavored Markdown:
  - `**bold**`, `*italic*`, `` `code` ``
  - Headings, lists, links, images, tables, blockquotes
  - Line breaks are preserved (press `Enter` for new paragraphs)
- **Automatic RTL/LTR Detection** ‚Äì The description side adjusts its text direction based on the presence of Arabic, Hebrew, or other right‚Äëto‚Äëleft scripts.
- **Responsive Design** ‚Äì Optimised for both desktop and mobile:
  - Collapsible sidebar with hamburger menu on small screens
  - Touch‚Äëfriendly buttons and cards
  - Adaptive grid layout
- **Dark Theme** ‚Äì Easy on the eyes, perfect for late‚Äënight studying.
- **Persistent Storage** ‚Äì All data is stored in a local `data.json` file (no database required).
- **Local Dependencies** ‚Äì All libraries (MathJax, marked, DOMPurify) are downloaded and served locally ‚Äì no CDN, no npm/Node.js needed.

## üöÄ Getting Started

### Prerequisites

- Python 3.7 or higher
- `pip` (Python package manager)

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/yourusername/flashcard-manager.git
   cd flashcard-manager
   ```

2. **Install Python dependencies**
   ```bash
   pip install flask requests
   ```

3. **Download frontend libraries**
   ```bash
   python download_libs.py
   ```
   This script will:
   - Download `marked.min.js` and `purify.min.js` from CDN.
   - Download and extract MathJax (version 3.2.2) into `static/js/mathjax/`.

4. **Run the server**
   ```bash
   python server.py
   ```

5. **Open your browser** and navigate to:
   ```
   http://localhost:5000
   ```

## üìñ Usage Guide

### Managing Categories
- The sidebar lists all existing categories.
- Click a category to view its flashcards.
- Enter a new category name in the input field and click **`+`** to create it.
- Click the **`‚úï`** next to a category to delete it (this removes all its cards as well).

### Working with Flashcards
- Once a category is selected, click **`+ Add Flashcard`** to open the modal.
- Fill in:
  - **Front** ‚Äì The question, term, or prompt.  
    Use `$...$` for LaTeX formulas (e.g., `$E=mc^2$`).
  - **Back** ‚Äì The answer or description.  
    Supports full Markdown (e.g., `**bold**`, `- list items`, `[links](url)`).
- Click **Save** ‚Äì the card will appear in the grid.
- Click any card to flip it (front ‚Üî back).
- Use the **`‚úé`** (edit) or **`‚úï`** (delete) buttons on each card to modify or remove it.

### LaTeX & Markdown Examples
| Front (with LaTeX) | Back (with Markdown) |
|-------------------|----------------------|
| `$ \int_0^\infty e^{-x} dx $` | `**Integral** ‚Äì converges to *1*.` |
| `$ \sum_{i=1}^n i = \frac{n(n+1)}{2} $` | `[Learn more](https://example.com)` |

> **Note:** If the entire back content is wrapped in `$...$`, it will be rendered as LaTeX instead of Markdown (useful for purely mathematical descriptions).

### RTL/LTR Auto‚ÄëDetection
- The back side detects characters from Arabic, Hebrew, Persian, and other RTL scripts.
- The appropriate CSS class (`rtl` or `ltr`) is applied automatically, ensuring correct text alignment.

## üì¶ API Endpoints (for developers)

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET    | `/api/categories` | List all categories |
| POST   | `/api/categories` | Create a new category |
| DELETE | `/api/categories/<name>` | Delete a category |
| GET    | `/api/cards/<category>` | Get all cards in a category |
| POST   | `/api/cards` | Add a new card |
| PUT    | `/api/cards/<id>` | Update a card (supports moving to another category) |
| DELETE | `/api/cards/<id>` | Delete a card |

All requests and responses are in **JSON** format.
